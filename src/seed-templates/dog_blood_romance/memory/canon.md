# 正史设定

本模板尚未绑定任何具体书籍正史。创建新书后,只有用户确认、章节正文已经写成、或 memory update 明确合并的事实才能进入正史。狗血言情 模板中的示例词、示例角色和示例冲突都不是新书事实。

## 变更记录

- 初始: 保持空正史,等待真实章节生成后再记录。
- 合并原则: 只记录已经发生且后续不可违背的事实。
- 禁止事项: 不把模板说明、agent 建议、未采纳大纲直接写成正史。

## 已确认事实

- 暂无。第一章完成并经过决策后再补充。

## 待确认事实

- 主角姓名、核心欲望、关键对手、长期冲突均需由具体项目决定。
## 维护说明

这段说明用于保证模板有足够可读信息。使用本模板起书时,请把它看作题材约束和质量底线,不要把其中的示例人物、示例事件或示例冲突直接当作新书正史。新书应当从用户概念、作者长期规划和后续章节反馈重新生成具体内容;模板只负责提醒哪些风险需要检查、哪些承诺不能丢失、哪些信息应当进入后续记忆。
